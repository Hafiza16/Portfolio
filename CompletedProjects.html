WORDLE IN PYTHON

As part of a Computing project, I had re-created the world-famous games of Wordle in Python using the Guizero library:

As you can see in the image on the right, the player is presented with a 5 by 6 grid in which they can enter 6 guesses of five letters. To do this, they need to enter their guess in the textbox input field and press the pink button.


If the user has the exact same letter in the same place, the letter goes green. However, if the user has the same letter but in a different place, the letter turns yellow, and if that letter is not in the word, it turns red.


Once the user guesses the word correctly before using all of their guesses, a message appears saying "Congrats" and they are not allowed to enter anymore guesses. Otherwise, the word is shown once the player has used all of the guesses and still has not got the word correct.


In the following screenshot, I am guessing a word to demonstrate the colour coding for the letters that occurs.

There are also messages to motivate the user to keep guessing, which are randomly selected from a string array to ensure they are different each time the player makes a guess.

Each time the user makes a guess, it fills in the next row of the grid with what they have entered (only entering the first five letters the player has input into the textbox, even if they have entered more letters.)



In this screenshot, a full game of Wordle is happening, with the word being correctly guessed after five attempts:











TEXT-BASED ADVENTURE

In Python, I had attempted to create a text-based adventure game similar to the online game Zork. These text-based games follow the simple basis of setting a storyline for the player and asking them to enter text-based inputs, allowing them to determine what happens next.

In the screenshot below, there is a short introduction for the player to read, allowing for them to understand the rules of and how to play the game:



The player is then required to enter a specific cardinal direction, resulting in a specific room that is entered.

From here, the player can choose areas to search for items, and can:

  • Pick up an item
  • Leave the item
  • Return to the main hallway
  • View their inventory

The player must choose an option to be able to progress onto the next step of the game.


Once the player has collected all of the items from all of the rooms that have been explored, the player enters into a battle with a mythical creature:

The player is reminded of the items in their inventory, and they must choose only three items from the inventory in order to enter the battle with the mythical creature.

However, there is a specific combination of items which will make the player successful in defeating the creature !


SPEEDY KEYS

For my A Level Computer Science Project, I had decided to create a game using C# for Unity Development.

The initial idea was to create a game in which random objects would appear on the screen at random times with an associated key and a time limit, and the user would have to press that key before the time ended in order to successfully save the object (and not lose a life).

However, I was unable to achieve this due to limitations of creating and instantiating too many objects in the game at a time, as spawing too many objects at the same time was causing the game to crash or become laggy, therefore I had to change my initial idea in order to reduce the effects of this.

Therefore, I decided to keep the initial game design the same, but the player had a character they could control using WASD keys and the aim was to collect as many coins as possible in order to exchange them to design their own character. As the levels went on, there were difficulties added e.g. having to avoid certain objects otherwise a life would be lost or having a boost.



The above images are screenshots of the game in action, with the player being able to collect coins, avoid enemies and create their own character in the drawing canvas.


STUDY SPACER

For my BTEC Computing project, I was tasked with creating an application to help students with planning their study and revision. I wanted to create an application that was available on both Android and iOS platforms, so I had developed this application using Thunkable, using Adobe XD for prototyping.



As you can see in the screenshot on the left, there is a loading screen with the app name "Study Spacer", which is shortly followed by an introduction screen.

The intro screen provides the user with an overview of the app and its purpose, allowing them to gain a better understanding of the app.

I have decided to use a panda image as the logo and a light blue for the background to provide the user with a calm and relaxing space in which they can map out their revision. This is because I feel that it is important for the student (the user) to be in this calm space in order to maximise their productivity when revising.

Once the user clicks the "Sounds Good" button, they are taken to a page with three options (on the right).
